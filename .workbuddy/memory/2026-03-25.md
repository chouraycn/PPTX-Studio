## 颜色跟随模板配色 + 版式套用修复（apply_template.py & beautify_ppt.py）

### apply_template.py（Mode 1）改动
1. **LAYOUT_TYPE_KEYWORDS 补中文关键词** — 新增"首页/封面/章节/两栏/图片/表格"等，解决中文模板版式名完全匹配失败问题
2. **`_detect_layout_type` 升级** — 增加占位符数量分析（body>=2 → two_column；无 body → section_header）
3. **新增 `_extract_template_colors`** — 从模板 theme XML 读取 dk1/lt1/accent1-3/dk2 色值，生成 primary/secondary/accent/bg/text 配色字典
4. **`_replace_placeholder_text` 重写** — 清空占位符内所有旧 `<a:p>` 节点，注入带模板颜色 `solidFill` 的新 run，解决旧颜色残留问题
5. **`_build_body_xml` 增加 `color` 参数** — bullet 列表的每个 run 携带 `solidFill` 强制使用模板正文颜色
6. **`_slide_has_dark_bg` 新函数** — 读取幻灯片背景色亮度，自动判断深色/浅色背景，决定文字用白色还是主色

### beautify_ppt.py（Mode 2）改动
1. **`_update_text_colors` 重写** — 原只替换10个硬编码颜色；改为：遍历所有 `<p:sp>`，title placeholder 内的 run 用 primary，其余用 text_on_light；只替换有 `solidFill` 的 run，继承色不动
2. **`_update_shape_colors` 优化** — luminance 阈值从 220 收紧到 235（近白），20 以下跳过（近黑），额外跳过模板自身 bg_light/bg_dark 颜色，其余全部换成 primary，覆盖更多中间色装饰元素

## Python 3.9 兼容性修复（方向 B）

三个脚本使用了 Python 3.10+ 才支持的 `str | None` 联合类型语法，在 Python 3.9 运行时报 TypeError。
采用方向 B（`from __future__ import annotations`），在以下三个文件 docstring 之后加入该导入，无需修改类型注解本身：
- `scripts/add_slide.py`
- `scripts/merge_pptx.py`
- `scripts/office/pack.py`
