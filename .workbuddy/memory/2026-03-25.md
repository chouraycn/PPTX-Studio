## 空白页填充完善 (2026-03-25 第十二轮)

### 根因分析
模板套用后出现空白页的根本原因有4个：

1. **`_create_slide_from_layout` 创建完全空白 slide**（最核心问题）
   - 当 template PPT 中有 layout 但没有使用该 layout 的 slide 时，
     `_find_template_slide_for_layout` 返回 None，退路走 `_create_slide_from_layout`
   - 旧实现创建的是一个完全空的 `<p:spTree>`，没有任何 placeholder 形状
   - 导致后续所有 `_replace_placeholder_text` / `_replace_placeholder_content` 找不到 `<p:ph>` 而全部无效

2. **`_replace_placeholder_content` 不支持无 lstStyle 的 placeholder**
   - 新创建的 placeholder 只有 `<a:bodyPr/><a:lstStyle/>`，但正则要求 `</a:lstStyle>` 后有内容
   - 导致新创建的 placeholder body 内容无法注入

3. **`_inject_content_into_slide` 缺 fallback 机制**
   - 当 title 注入失败（layout 没有对应 ph 类型），整个 slide 保持为空

4. **`extract_content.py._extract_body_text` 把 subTitle/dt 也当 body 提取**
   - 导致 title_slide 类型的 subtitle 内容同时出现在 subtitle 和 body 字段中，造成重复注入和逻辑混乱

### 修复内容
1. **新增 `_extract_layout_placeholders()`** — 从 layout XML 提取所有 placeholder 的：
   - ph_type（title/body/subTitle/ctrTitle），ph_idx，shape id，shape name
   - xfrm 位置信息（x/y/cx/cy）；若 layout 未定义则用合理的默认值

2. **新增 `_build_placeholder_sp_xml()`** — 从 placeholder 定义构建最小 `<p:sp>` XML：
   - 包含完整的 `<p:nvSpPr>/<p:ph type="...">` 标签（让后续注入能匹配到）
   - 包含 `<a:xfrm>` 位置信息
   - `<a:txBody>` 只有 `<a:bodyPr/>` + `<a:lstStyle/>` + 空段落（等待注入）

3. **重写 `_create_slide_from_layout()`** — 读取 layout XML → 提取 placeholder → 在新 slide 中预填充对应形状

4. **升级 `_replace_placeholder_content()`** — 新增 Case 2 fallback：当 txBody 无 lstStyle 时，保留 bodyPr 并替换所有 `<a:p>` 段落

5. **升级 `_inject_content_into_slide()`** — 新增 title 注入失败的 fallback：尝试注入到 body/obj/subTitle

6. **修复 `extract_content.py._extract_body_text()`** — 将 subTitle/dt/ftr/sldNum 从 body 提取中排除，只提取 body/obj 类型和无类型(idx-only)占位符

## 模板样式深度自动分析与适配 (2026-03-25 第十一轮)

### apply_template.py 深度模板样式分析

**需求**：模板套用后 PPT 应自动分析模板的完整样式并全面适配，不只是颜色。

**新增/升级内容**：
1. **新增 `_extract_template_fonts()`** — 从 `ppt/theme/theme1.xml` 的 `<a:fontScheme>` 提取：
   - `major_latin`：标题字体（Latin/西文）
   - `major_ea`：标题字体（East Asian/CJK）
   - `minor_latin`：正文字体（Latin）
   - `minor_ea`：正文字体（CJK）
   - 空字符串 = 不设置，让 PowerPoint 继承 layout/master（安全兜底）

2. **新增 `_extract_layout_placeholder_styles()`** — 从每个 slideLayout XML 提取：
   - `bodyPr_attrs`：placeholder 的 bodyPr 属性（边距、wrap、anchor 等）
   - `defPPr_xml`：默认段落属性（spacing 等）
   - `lvl1pPr_xml`：一级段落属性
   - `defRPr_xml`：默认 run 属性（字号等）
   - `default_sz`：解析出的默认字号（int）

3. **升级 `_analyze_template_layouts()`** — 现在为每个 layout 额外返回 `ph_styles` 字段（每个 placeholder 类型对应的样式信息）

4. **`apply_template()` 主函数** — 新增调用 `_extract_template_fonts()` 并传给 `_build_new_slides`

5. **升级 `_build_new_slides()`** — 签名加入 `template_fonts: Dict[str, str]`，从 `layout_lookup` 取出 `layout_ph_styles` 传给 `_inject_content_into_slide`

6. **升级 `_inject_content_into_slide()`** — 接收 `template_fonts` 和 `layout_ph_styles`；标题用 `major_latin/major_ea`，正文用 `minor_latin/minor_ea`

7. **升级 `_replace_placeholder_text()`** — 新增 `latin_font` / `ea_font` 参数；写入 `<a:latin typeface="..."/>` 和 `<a:ea typeface="..."/>`；**修复 fallback 分支**：不再用空 `<a:bodyPr/>`，改为保留原有 `<a:bodyPr>` 内容

8. **升级 `_build_body_xml()`** — 新增 `latin_font` / `ea_font` 参数；构建 `rpr_children = colour + latin + ea`

**适配规则完整版（2026-03-25 第十一轮最终）**：
| 项目 | 行为 |
|------|------|
| 文字颜色 | 模板配色（primary / text_on_light / text_on_dark） |
| Latin 字体 | 模板 fontScheme（major=标题/minor=正文），空=继承 |
| CJK 字体 | 模板 fontScheme（major_ea/minor_ea），空=继承 |
| 加粗/斜体 | 从 body_rich 保留 |
| 字号 | 从 body_rich 保留 |
| 项目符号 | 继承 layout lstStyle |
| 语言标签 | 自动检测（zh-CN/>10% CJK，否则 en-US） |
| 背景判断 | srgbClr / schemeClr / bgRef 四层检测 |
| bodyPr（边距/wrap） | 保留模板 placeholder 的设置，不覆盖 |
